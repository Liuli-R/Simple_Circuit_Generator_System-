# Simple Circuit Generator System

## 项目简介

Simple Circuit Generator System 是一个基于 Qt 6 和 C++ 的简易电路图生成系统。项目当前进入 **v1 阶段版**：在 v0.5 已完成“元器件图形生成与界面结构整理”的基础上，进一步加入了自动串联闭合回路、导线绘制、运行后状态刷新，并对 `MainWindow` 做了职责拆分。

当前版本仍然以“简单串联电路”为主要目标。手动连线、并联结构、复杂拓扑求解等内容暂未作为 v1 的核心范围。

## v0.5 到 v1 的主要变化

### v0.5 已完成内容

v0.5 主要完成了电路图生成系统的图形基础：

- 支持小灯泡、电源、开关、电阻、电流表、电压表等元器件图形绘制
- 元器件图形统一继承 `ComponentItem`，复用可移动、可选中、端点绘制、边界检测等能力
- 使用 `CircuitScene` 绘制网格画布背景
- 通过 `ComponentPaletteWidget` 封装左侧元器件栏
- 点击左侧按钮后自动创建模型对象和对应图形项
- 初步整理了 `Graphics/items`、`Graphics/manager`、`Graphics/scene` 等目录结构

这一阶段的重点是“能把元器件放到画布上”，尚未处理导线连接和运行后的电路状态表现。

### v1 新增内容

v1 在 v0.5 的基础上补上了一个简单串联电路的基本运行链路：

- `Circuit::buildClosedLoop()` 按元器件添加顺序自动构造主串联闭合回路
- 点击运行后自动清除旧导线并重新绘制串联导线
- 新增 `WireManager`，专门负责导线图形的清理与绘制
- 新增 `ComponentAddController`，专门负责元器件模型与图形项的创建
- 开关图形支持双击切换状态，并通过回调同步到电路模型
- 运行后通过 `CircuitSolver` 更新灯泡、电流表、电压表等模型状态
- `MainWindow` 从“什么都管”调整为整体流程协调者，代码行数降到 200 行以内
- `CMakeLists.txt` 已纳入新增的应用层和导线管理类

可以理解为：v0.5 解决“画出来”，v1 开始解决“按简单串联方式连起来并运行”。

## 当前运行流程

当前 v1 的运行流程大致如下：

1. 用户点击左侧元器件按钮
2. `ComponentAddController` 创建模型对象并交给 `Circuit`
3. `ComponentItemManager` 创建对应图形项并加入画布
4. 用户点击运行按钮
5. `Circuit` 按添加顺序构造简单串联闭合回路
6. `WireManager` 根据图形端点绘制串联导线
7. `CircuitSolver` 计算电路状态
8. `MainWindow` 刷新灯泡等图形显示

## 当前项目结构

```text
.
|-- Application/
|   `-- ComponentAddController.*   # 元器件添加流程控制
|-- Circuit_code/                   # 电路元件、节点、连接、电路模型和求解逻辑
|-- Graphics/
|   |-- items/                      # 各类 QGraphicsItem 元器件图形
|   |-- manager/                    # 图形项管理与导线管理
|   `-- scene/                      # 自定义画布场景
|-- componentpalettewidget.*        # 左侧元器件库自定义控件
|-- mainwindow.*                    # 主窗口、场景装配和运行流程协调
|-- main.cpp                        # Qt 应用入口和全局样式加载
|-- res.qrc                         # Qt 资源文件
|-- style.css                       # 界面样式
`-- CMakeLists.txt                  # CMake 构建配置
```

## 主要类说明

### `MainWindow`

主窗口现在主要负责界面装配和流程协调，包括初始化场景、设置运行按钮、连接信号槽、运行电路、清空电路以及刷新图形状态。

### `ComponentPaletteWidget`

左侧元器件库控件，负责按钮、图标、标题栏和对外添加请求信号。

### `ComponentAddController`

应用层控制类，负责添加元器件时的完整流程：

- 创建具体元器件模型
- 添加到 `Circuit`
- 为主回路元器件添加节点
- 创建对应图形项
- 为开关图形绑定状态同步回调

### `ComponentItemManager`

图形元器件管理类，负责根据模型元件类型创建对应的图形项，并根据元器件 id 查找图形项。

### `WireManager`

导线图形管理类，负责清除旧导线和根据当前主串联元器件顺序绘制导线。当前导线使用 `QGraphicsLineItem` 实现，适合作为 v1 的轻量方案。

### `Circuit`

电路模型类，保存元器件与节点，并提供自动构造简单串联闭合回路、判断闭合回路、查找元器件等功能。

### `CircuitSolver`

简单串联电路求解类，负责根据当前电路状态更新电流表、电压表和灯泡状态。

## 构建环境

项目使用 CMake 管理构建，依赖 Qt 6：

- CMake 3.19 或更高版本
- Qt 6.5 或更高版本
- C++ 编译环境

推荐通过 Qt Creator 打开项目并构建运行。

## 当前限制与后续方向

v1 目前只支持自动串联结构，导线绘制也主要基于元器件图形端点之间的直线连接。后续可以继续推进：

- 手动拖拽连线
- 并联和复杂拓扑结构
- 图形端点与 `Node` 坐标的真实同步
- `Connection` 类与手动连线流程的整合
- 电压表测量目标的可视化绑定
- 电流表、电压表读数的图形端显示
- 运行结果弹窗、状态栏提示和错误提示优化
- 窗口缩放下的动态画布边界处理

整体来说，v1 已经把“元器件生成、自动串联、可视化导线、运行状态刷新”串成了一个基础闭环，后续重点会转向更真实的连线交互和电路拓扑表达。
