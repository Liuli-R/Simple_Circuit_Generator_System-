# Simple Circuit Generator System

## 项目简介

Simple Circuit Generator System 是一个基于 Qt 6 和 C++ 的简易电路图生成系统。当前版本定位为 **v0.5 正式版**，重点完成了从“静态界面骨架”到“可点击生成图形元器件”的阶段性闭环。

v0.5 版本暂未处理导线绘制与真实连线关系，当前主要围绕元器件图形实例化、场景绘制、边界控制、界面结构拆分和工程结构整理展开，为后续导线连接、电路拓扑编辑和运行状态同步打基础。

## v0.5 完成内容

### 1. 图形绘制与重绘

项目已完成主要元器件的 `QGraphicsItem` 图形绘制，包括小灯泡、电源、开关、电阻、电流表和电压表。

各类图形项通过继承 `ComponentItem` 复用通用能力，例如元器件编号、可移动、可选中、端点绘制和端点坐标映射。具体图形由各自的 `paint()` 函数完成，加入 `QGraphicsScene` 后由 Qt 自动触发绘制与重绘。

### 2. 画布边界检测

`ComponentItem` 增加了位置变化处理逻辑，用于防止元器件被拖出画布范围。当前 v0.5 采用固定画布/固定窗口思路，边界主要基于 `sceneRect` 进行限制。

后续版本可以考虑根据 `QGraphicsView` 的可见区域动态调整边界，使画布范围随窗口视图变化而变化。

### 3. 按钮与元器件自动实例化

左侧元器件按钮通过 `QAction` 与信号槽机制连接到主窗口逻辑。点击按钮后，会自动完成：

- 创建对应的电路模型对象
- 添加到 `Circuit` 中
- 为主回路元器件创建节点
- 创建对应的 `ComponentItem`
- 将图形项加入画布并自动摆放

电压表目前作为测量元件处理，暂不自动加入主串联回路节点。

### 4. 左侧栏自定义控件封装

原本放在 `MainWindow` 中的左侧元器件栏逻辑，已拆分为 `ComponentPaletteWidget` 自定义控件。

该控件负责元器件库标题、按钮创建、图标显示和内部信号转发。`MainWindow` 不再直接管理左侧栏内部布局，只接收 `ComponentPaletteWidget` 发出的添加请求，从而让主窗口更专注于场景、电路模型和整体流程调度。

### 5. 工程结构整理

v0.5 对图形相关代码进行了目录整理，并清理了部分多余 include，使工程结构更清晰：

- 元器件图形项集中到 `Graphics/items`
- 图形项管理类集中到 `Graphics/manager`
- 场景绘制类集中到 `Graphics/scene`
- 电路模型和求解逻辑继续保留在 `Circuit_code`
- 左侧自定义控件目前保留在项目根目录，后续也可继续整理到 `Widgets`

## 新增与调整的主要类

### `ComponentPaletteWidget`

左侧元器件库控件，负责管理元器件按钮和对外发出添加请求信号。

当前包含：

- 元器件 QAction 创建
- 元器件按钮创建
- 元器件库标题栏样式设置
- 按钮点击信号转发

### `ComponentItemManager`

图形元器件管理类，负责根据电路模型元件创建对应的 `ComponentItem` 子类，并加入 `QGraphicsScene`。

当前包含：

- 根据元件类型创建对应图形项
- 设置图形项默认位置
- 保存图形项指针记录
- 根据元件 id 查找图形项
- 清空画布图形项记录

### `ComponentItem`

所有图形元器件的基类，提供可移动、可选中、端点绘制、端点坐标映射和画布边界检测等通用能力。

### `CircuitScene`

自定义电路场景，负责绘制浅绿色网格背景，为后续元器件对齐、导线绘制和画布交互提供基础。

## 当前项目结构

```text
.
|-- Circuit_code/              # 电路元件、节点、连接、电路模型和求解逻辑
|-- Graphics/
|   |-- items/                 # 各类 QGraphicsItem 元器件图形
|   |-- manager/               # 图形项管理
|   `-- scene/                 # 自定义画布场景
|-- Cpp_Practice_Picture/      # 元器件图标和界面资源图片
|-- componentpalettewidget.*   # 左侧元器件库自定义控件
|-- mainwindow.*               # 主窗口、场景装配和整体流程调度
|-- main.cpp                   # Qt 应用入口和全局样式加载
|-- res.qrc                    # Qt 资源文件
|-- style.css                  # 界面样式
`-- CMakeLists.txt             # CMake 构建配置
```

## 构建环境

项目使用 CMake 管理构建，依赖 Qt 6：

- CMake 3.19 或更高版本
- Qt 6.5 或更高版本
- C++ 编译环境

推荐通过 Qt Creator 打开项目并构建运行。

## 当前限制与后续方向

v0.5 版本尚未完成导线绘制和真实连线关系处理。当前元件可以被添加到画布并参与基础模型记录，但图形端点之间尚未形成可视化导线，也没有实现手动连接、并联分支或复杂拓扑求解。

后续版本可优先考虑：

- 导线图形项与导线管理类
- 端点点击、拖拽连线和吸附
- 图形连线与 `Node` 节点关系同步
- 电压表测量目标绑定
- 运行后灯泡、表计读数与图形状态同步
- 窗口尺寸变化下的动态画布边界处理
