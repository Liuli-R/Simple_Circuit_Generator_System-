# Simple Circuit Generator System

基于 Qt 6 与 C++ 开发的简易电路图生成、连通检测与串联电路演示工具。

当前版本为 **v1.0.0**。项目已经从 v0.5 的“元器件能够创建并绘制到画布”发展为具备元器件管理、自动串联拓扑、蛇形布局识别、直角导线绘制、电路求解、参数设置和运行状态反馈的完整演示流程。

> v1.0.0 的重点是打通一条清晰、可运行的简单串联电路链路。手动连线、并联电路和任意布局避障尚未纳入当前版本的核心范围。

## 发行版本所支持测试环境

- 支持在win10（虚拟机进行测试成功）、win11运行，同时支持在Ubuntu 24.04.3（虚拟机进行测试成功）运行！

## 版本概览

### v0.5：图形基础版本

v0.5 完成了电路绘图系统的基本框架，主要解决“元器件如何出现在画布上”的问题：

- 支持电源、灯泡、开关、定值电阻、电流表和电压表的图形绘制。
- 各类图形项统一继承 `ComponentItem`，复用移动、选择、端点和边界检测能力。
- 使用 `CircuitScene` 绘制固定网格背景。
- 使用 `ComponentItemManager` 创建、保存和查找图形项。
- 点击左侧元器件库后，能够同步创建模型对象与图形对象。
- 元器件可以拖动，但不能移出 `900 × 650` 的场景边界。
- 左侧元器件库从 `MainWindow` 中拆出，封装为 `ComponentPaletteWidget`。
- 初步整理 `Graphics/items`、`Graphics/manager` 和 `Graphics/scene` 等目录。
- 清理多余 `include`，统一主要代码格式和工程结构。

这一阶段尚未形成真正的可视化电路：图形之间没有导线，运行按钮也没有完整串起拓扑构建、计算与状态更新。

### v1.0.0：自动串联运行版本

v1.0.0 在 v0.5 的图形基础上补齐了模型、布局、连线、计算和界面反馈：

- 新增 `ComponentAddController`，统一负责模型和图形项的创建。
- 新增 `CircuitLayoutBuilder`，在运行前读取画布中主回路元器件的实际位置。
- 根据纵坐标完成分行，并按奇偶行生成蛇形排列顺序。
- `Circuit::buildClosedLoop()` 根据布局顺序重新建立主回路节点连接。
- 新增 `WireManager`，统一管理主回路导线和电压表测量支路。
- 导线由简单直线升级为 `QPainterPath + QGraphicsPathItem` 直角折线。
- 支持沿左右安全通道换行，并沿外围完成最后一段闭合连接。
- 电压表不进入主串联回路，可绑定指定元器件并显示对应测量支路。
- `CircuitSolver` 返回结构化的 `CircuitSolveResult`，不再只表达单一成功或失败状态。
- 支持未运行、回路断开、缺少电源、无有效电阻和正常运行等状态。
- 支持多个电压表结果，状态 Dock 可通过下拉框切换查看。
- 开关支持双击切换，并在首次运行后触发电路状态实时刷新。
- 新增电路参数设置窗口，可设置电源电压、电阻阻值和电压表测量目标。
- 新增右侧 `CircuitStatusDock`，显示回路状态、总电压、总电阻、电流和电压表读数。
- 新增运行、清空、设置和使用说明等菜单操作及快捷键。
- 新增说明窗口，并通过 Markdown 资源展示使用指南。
- 全局样式由单一文件拆分为多个 QSS 模块，界面统一为暖色工具风格。
- `Circuit` 的拓扑实现拆入 `CircuitTopology.cpp`，利用 C++ 分离编译控制文件职责和规模。

可以将两个版本的变化概括为：

```text
v0.5：创建元器件并画出来
  ↓
v1：识别布局 → 建立拓扑 → 绘制导线 → 求解电路 → 更新界面
```

## 当前支持的元器件

| 元器件 | 模型作用 | 图形与交互 |
| --- | --- | --- |
| 电源 `Battery` | 提供直流电压，多个电源电压相加 | 可拖动，电压可在设置窗口修改 |
| 灯泡 `Bulb` | 作为有电阻的用电元件 | 电路正常运行时点亮，否则熄灭 |
| 开关 `Switch` | 控制主回路导通状态 | 双击切换开合状态 |
| 定值电阻 `Fixed_resistor` | 参与总电阻计算 | 可选择指定电阻并修改阻值 |
| 电流表 `Ammeter` | 记录主串联回路电流 | 状态面板显示测量结果 |
| 电压表 `Voltmeter` | 测量目标元器件两端电压 | 绑定目标后绘制并联测量支路 |

## 使用方法

### 1. 添加元器件

从左侧元器件库点击需要的元器件。系统会同时：

1. 创建对应的 C++ 模型对象。
2. 将模型交给 `Circuit` 管理。
3. 为主回路元器件创建节点。
4. 创建对应的 `ComponentItem` 并放入场景。

电压表是测量支路，不会额外占用主回路节点。

### 2. 调整画布布局

v1.0.0 使用受约束的蛇形自动布线。推荐按以下顺序摆放：

```text
第一行：A → B → C
                  ↓
第二行：F ← E ← D
        ↓
沿左侧安全通道返回第一项
```

布局分析的主要规则：

- 按元器件中心纵坐标分行。
- 当前行判断容差为 `80px`。
- 第 0、2、4 行从左向右排序。
- 第 1、3、5 行从右向左排序。
- 行末通过左右安全通道进入下一行。
- 最后一个元器件沿外围连接回第一个元器件。

为了得到更整齐的导线，应避免元器件互相覆盖，并在画布四周留出一定的走线空间。

### 3. 设置电路参数

通过菜单栏的“设置”打开电路参数窗口，可进行：

- 修改当前电源和后续新建电源的默认电压。
- 选择一个定值电阻并设置阻值。
- 选择电压表及其测量目标。

完成电压表设置后，图形会自动移动到目标元器件上方。若画布中存在电压表，建议在运行前先完成测量目标设置。

### 4. 运行电路

点击“运行”后，程序依次执行：

1. `CircuitLayoutBuilder` 读取主回路图形端点和边界。
2. 根据画布位置生成 `CircuitLayout` 快照。
3. `Circuit` 按布局顺序分配节点并构建闭环。
4. 电压表根据 `targetComponentId` 同步目标节点。
5. `WireManager` 清除旧导线并绘制新的主回路与测量支路。
6. `CircuitSolver` 检查开关、闭环、电源和有效电阻。
7. 求解器计算总电压、总电阻、电流及各电压表读数。
8. 状态 Dock 和灯泡图形刷新。

### 5. 查看运行结果

右侧状态 Dock 显示：

- 电路运行状态。
- 总电压。
- 总电阻。
- 电流表示数或缺失提示。
- 电压表选择、测量目标和读数。

存在多个电压表时，可使用下拉框切换查看各自结果。

### 6. 操作开关

双击开关图形即可切换开合状态。电路至少运行过一次后，开关变化会触发重新求解，并同步更新灯泡和状态 Dock。

### 7. 清空画布

“清空”操作会同步处理：

- 主回路导线和电压表支路。
- `Circuit` 中的元器件与节点。
- 场景中的全部元器件图形。
- 元器件 ID 与节点 ID 计数。
- 求解结果、状态 Dock 和实时更新标记。

## v1.0.0 电路计算规则

当前求解器面向简单直流串联电路：

```text
总电压 U = 所有电源电压之和
总电阻 R = 主回路中非电源、非电压表元器件电阻之和
主回路电流 I = U / R
目标元器件电压 = 目标电阻 × I
```

以下情况不会进入正常运行状态：

- 任一开关断开。
- 主回路没有形成逻辑闭环。
- 缺少电源。
- 总电阻小于或等于零。

## 运行流程与数据关系

```text
ComponentPaletteWidget
        │ 添加请求
        ▼
ComponentAddController
        ├─ 创建 Component 模型
        └─ 创建 ComponentItem 图形
                 │
                 ▼
        CircuitLayoutBuilder
                 │ CircuitLayout
          ┌──────┴────────┐
          ▼               ▼
CircuitTopology       WireManager
建立逻辑节点连接       绘制可视化导线
          │               │
          └──────┬────────┘
                 ▼
           CircuitSolver
                 │ CircuitSolveResult
          ┌──────┴────────┐
          ▼               ▼
  CircuitStatusDock    ComponentItem
    显示计算结果         更新灯泡状态
```

`CircuitLayout` 是每次运行前生成的图形快照。它只保存当前端点、边界、行号和排序结果，不改变 `Circuit` 与 `ComponentItemManager` 原有容器的所有权顺序。

## 主要模块职责

### `MainWindow`

负责创建场景和各管理对象、连接菜单与控件信号、组织运行和清空流程。具体元器件创建、布局分析、导线绘制与电路计算分别交给独立模块处理。

### `ComponentAddController`

负责添加元器件时的完整应用流程，包括模型创建、主回路节点创建、图形项创建、默认参数传递和开关状态回调。

### `ComponentItemManager`

根据模型类型创建对应图形项，保存场景中的 `ComponentItem` 指针，并通过稳定的元器件 ID 完成模型与图形查找。

### `CircuitLayoutBuilder`

运行前扫描主回路元器件的真实场景坐标，按纵坐标分行、按横坐标排序，并生成蛇形 `CircuitLayout`。布局数据是临时快照，不替换原始容器。

### `WireManager`

使用 `QPainterPath` 描述多段直角路径，再通过 `QGraphicsScene::addPath()` 创建 `QGraphicsPathItem`。同时管理主回路导线和电压表并联测量支路。
os:优化了导线的排布问题原先由Line变为Path折线更贴近绘制电路图的风格以及尽量避免产生斜线交叉

### `Circuit` 与 `CircuitTopology.cpp`

`Circuit` 持有元器件与节点。拓扑实现根据布局 ID 顺序分配左右节点，通过取模使最后一个元器件返回起始节点，并在重建主回路后重新同步已绑定电压表的节点。

### `CircuitSolver`

检查回路状态并计算简单串联电路。结果统一写入 `CircuitSolveResult`，界面层无需重新遍历模型推断每种显示状态。

### `CircuitStatusDock`

将结构化求解结果转换为用户可读文本，并使用 `QSignalBlocker` 安全刷新多个电压表下拉条目，避免程序更新控件时触发不必要的连锁信号。
------>QSignalBlocker的类使用由ai辅助提供代码学习

## 构建环境

- CMake `3.19+`
- Qt `6.5+`
- 支持 C++17 的编译器
- Qt 模块：`Core`、`Widgets`

推荐直接使用 Qt Creator 打开根目录的 `CMakeLists.txt`，选择可用的 Qt 6 Desktop Kit 后进行配置和构建。

也可以使用命令行：

```bash
cmake -S . -B build -DCMAKE_PREFIX_PATH="<Qt 安装目录>"
cmake --build build
```

Qt 图片、GIF、SVG、Markdown 和 QSS 均通过 `res.qrc` 打包。程序启动时会依次读取多个 QSS 资源并合并为全局样式表。

## 当前限制

v1.0.0 是受约束的自动串联版本，目前存在以下边界：

- 只支持简单直流串联主回路，不支持并联和复杂图拓扑求解。
- 尚未支持用户从端点拖拽并手动创建导线。
- 自动导线依赖蛇形摆放规则，不是通用障碍物寻路算法。
- 行判断采用固定 `80px` 容差，极端错位可能产生误分行。
- 外围安全通道需要用户在画布边缘保留一定空间。
- 电压表测量目标由用户在设置窗口中指定，而不是通过手动接线识别。
- 尚未加入撤回最近操作、删除单个元器件和操作历史。
- 场景与主窗口当前采用固定尺寸，暂未跟随 View 缩放动态扩展边界。
- `connection` 模型已预留，但尚未进入当前自动串联主流程。
